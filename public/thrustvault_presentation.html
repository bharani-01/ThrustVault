Project Vision & Objectives

Background Context

The Engineering Challenge

  • UAV powertrain calibration data is highly fragmented across testing labs.
  • Rig-generated data (e.g. Rotrix) is nested, containing merged headers and multi-run tests.
  • Manual copy-pasting of steps leads to human transcription errors.
  • Lack of comparison tools makes it difficult to run propeller efficiency optimizations.

The ThrustVault Solution

  • Centralized Database: A single secure repository mapping motors and test runs.
  • Curves Visualizer: Dynamic tool comparing multi-propeller calibration curves.
  • Smart Ingestion: Interactive uploader that pre-resolves merges and maps headers.
  • Access Security: Role-based boundaries protecting catalog databases.

Architecture & Tech Stack

System Layout

Database Layer

Supabase PostgreSQL

  • Relational tables tracking profiles, categories, motors, runs, and points.
  • Foreign Key cascades for consistent deletions.
  • JSONB extra_data column for flexible schema.

Backend

Flask Web Services

  • Static server hosting the lightweight client application.
  • Secure environment configuration bridging (/api/config).
  • Simple, zero-maintenance local runtime.

Ingestion

SheetJS Library

  • Client-side binary XLSX / XLS table reading.
  • HTML-table-based XLS spreadsheet support.
  • Merges resolving pass to fill-down spanning cells.

Frontend

Vanilla JS & Chart.js

  • Fluid Single Page Application (SPA) loading.
  • Comparative multi-propeller line chart plots.
  • Dynamic metric selection (Thrust, Efficiency, Current, RPM).

UAV Motor Catalog

Catalog Management

Key Catalog Features

  • Motors are structured under thrust class categories (e.g. 5kg Class).
  • Tracks precise motor specs: KV rating, Max Current, Max Thrust, Weight.
  • Features real-time search, sorting, and dynamic category listing.
  • Action Protection: CRUD forms are restricted to Admin/User profiles; Guests are locked into read-only.

Database Consistency

Database schema implements Foreign Key cascades. Deleting a Category or Motor automatically cleans up all child Test Runs and Data Points, avoiding orphan rows and database corruption.

Sidebar counters automatically update stats (Total Runs, Recorded Steps) upon edits.

Calibration Curves Visualizer

Comparative Analytics

Multi-Propeller Comparison

Visualizer overlays calibration curves for multiple propellers on a single interactive plot.

Allows engineers to identify the optimal propeller size for desired thrust ranges and efficiency targets.

Dynamic tooltips show exact metrics (e.g. 11.2 g/W at 1200g thrust).

Interactive Y-Axis Metrics

  • Thrust (g) vs. Throttle (%) — Verifies linear response.
  • Efficiency (g/W) vs. Thrust (g) — Models battery life and flight ranges.
  • Current (A) vs. Throttle (%) — Informs ESC and wiring limits.
  • RPM vs. Throttle (%) — Validates propeller safety margins.

Advanced Dataset Creator

Smart Ingestion Engine

Spreadsheet Ingestion & Grouping

  • Trims and decodes legacy `.xls` HTML table files, preventing CSV parsing errors.
  • Resolves merged headers/types (rowspan merges) automatically.
  • Groups rows by motor KV & propeller, displaying a **Multi-Run Selector** (e.g. splits 7 propeller tests in a single file).
  • Supports clipboard copy-pasting (Ctrl+V) from Excel and Copy-Down tools.

Interactive Column Mapper Modal

  • Opens a preview workspace for every uploaded file.
  • Dropdown selects allow mapping columns to system fields, metadata, or ignoring them.
  • Custom Columns: Adds dynamic columns (e.g. vibration) saved to the database inside the JSONB `extra_data` column.
  • Validation: Ensures Throttle and Thrust are mapped, and blocks duplicated or empty column name mappings.

Security & Access Control

Enterprise Protection

Role-Based Access Control (RBAC)

  • Secured by Supabase native authentication with JSON Web Tokens (JWT).
  • User profile roles dictate dashboard access:
    • Admin: Full CRUD catalog & database management.
    • User: Ingestion of test run datasets.
    • Guest: Read-only curve plots & visualizer.
  • Stores global user activity logs in localStorage.

Forgot Password OTP Flow

  • A multi-view workflow contained directly inside the login card.
  • Uses Supabase passwordless OTP: requests a secure 6-digit code via email.
  • Verifies the code on-screen to create a temporary session.
  • Allows resetting the password securely, then automatically log out to ensure fresh credentials.

Project Impact & Roadmap

Conclusion

Key Engineering Impact

  • Time Reductions: Data ingestion times reduced from minutes to single-click uploads.
  • Error Prevention: Schema constraints and validators block transcription errors.
  • Centralization: Historical curves are saved and searchable by engineers.
  • Robustness: Handles complex legacy test sheet structures cleanly.

Future Roadmap

  • Multi-Motor Overlay: Comparing performance curves of different motors together.
  • Export Services: Download formatted XLSX reports and PDF curves.
  • Hardware Ingestion: Connect directly to thrust test stand hardware via Bluetooth / Serial API for real-time live charting.
